# PWA Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Slicerex installable and fully functional offline after first visit.

**Architecture:** Use `vite-plugin-pwa` with Workbox `generateSW` strategy to auto-generate a service worker at build time. Add a web app manifest and placeholder icons. All static assets get precached — no runtime caching or offline page needed.

**Tech Stack:** vite-plugin-pwa, Workbox (via plugin), SVG icons converted to PNG

### Task 2: Create placeholder icons

**Files:**
- Create: `public/icon-192.png`
- Create: `public/icon-512.png`
- Create: `public/icon.svg`

- [ ] **Step 1: Create SVG icon source**

Create `public/icon.svg`:

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 512 512">
  <rect width="512" height="512" rx="64" fill="#16a34a"/>
  <text x="256" y="300" text-anchor="middle" font-family="sans-serif" font-weight="bold" font-size="200" fill="white">Sx</text>
</svg>
```

- [ ] **Step 2: Generate PNG icons from SVG**

```bash
npx sharp-cli -i public/icon.svg -o public/icon-192.png resize 192 192 && npx sharp-cli -i public/icon.svg -o public/icon-512.png resize 512 512
```

If `sharp-cli` is not available, use this Node.js one-liner:

```bash
node -e "
const sharp = require('sharp');
(async () => {
  const svg = await sharp('public/icon.svg').png().toBuffer();
  await sharp(svg).resize(192,192).png().toFile('public/icon-192.png');
  await sharp(svg).resize(512,512).png().toFile('public/icon-512.png');
})();
"
```

If sharp is not available either, create PNGs using canvas in a small script. The fallback is to create simple solid-color PNGs:

```bash
node -e "
const { createCanvas } = require('canvas');
for (const [name, size] of [['icon-192',192],['icon-512',512]]) {
  const c = createCanvas(size, size);
  const ctx = c.getContext('2d');
  ctx.fillStyle = '#16a34a';
  ctx.fillRect(0, 0, size, size);
  ctx.fillStyle = 'white';
  ctx.font = 'bold ' + (size * 0.4) + 'px sans-serif';
  ctx.textAlign = 'center';
  ctx.textBaseline = 'middle';
  ctx.fillText('Sx', size/2, size/2);
  require('fs').writeFileSync('public/' + name + '.png', c.toBuffer('image/png'));
}
"
```

If none of those work, create minimal valid PNGs manually:

```bash
node -e "
const { writeFileSync } = require('fs');
const { createCanvas } = require('canvas');
" 2>/dev/null || echo "Will use SVG icons in manifest instead"
```

If no PNG generation tool is available, use SVG icons directly in the manifest (works in modern browsers):

```json
{
  "src": "icon.svg",
  "sizes": "any",
  "type": "image/svg+xml"
}
```

### Task 4: Configure vite-plugin-pwa

**Files:**
- Modify: `vite.config.ts`

- [ ] **Step 1: Add PWA plugin to Vite config**

Replace the contents of `vite.config.ts` with:

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  base: '/slicerex',
  plugins: [
    react(),
    tailwindcss(),
    VitePWA({
      registerType: 'autoUpdate',
      manifest: false,
      workbox: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg,webmanifest}'],
      },
    }),
  ],
  worker: {
    format: 'es',
  },
})
```

Key decisions:
- `manifest: false` — we provide our own manifest file in `public/` rather than letting the plugin generate one, so we have full control
- `registerType: 'autoUpdate'` — new versions apply automatically on next visit
- `globPatterns` — precache all built assets including the worker and manifest

- [ ] **Step 2: Verify config is valid TypeScript**

Run: `npx tsc --noEmit vite.config.ts 2>&1 || npx tsc -b --dry 2>&1 | head -5`
Expected: no errors related to vite.config.ts

### Task 6: Register service worker in main.tsx

**Files:**
- Modify: `src/main.tsx`

- [ ] **Step 1: Add SW registration import**

Replace the contents of `src/main.tsx` with:

```typescript
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import { registerSW } from 'virtual:pwa-register';
import './index.css';
import App from './App.tsx';

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>
);

registerSW({ immediate: true });
```

- [ ] **Step 2: Add type declaration for virtual module**

Create `src/vite-pwa.d.ts`:

```typescript
declare module 'virtual:pwa-register' {
  export interface RegisterSWOptions {
    immediate?: boolean;
  }
  export function registerSW(options?: RegisterSWOptions): (reloadPage?: boolean) => Promise<void>;
}
```

Note: `vite-plugin-pwa` provides its own types, but adding this declaration ensures the import resolves if the plugin types aren't picked up automatically.
